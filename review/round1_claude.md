# Claude Round 1 리뷰 (독립)

> **청중 재확인**: NetLearning 회장 (일본 대학 LMS 1티어, 글로벌 AI LMS 트렌드 정통) + 조용상 교수 (KERIS 출신, CASE/EDU-API/CLR 국내 최고 권위).
> 이 청중은 "AI 붙였다"류의 뜬소리를 즉시 판별함. 이 문서가 그 테스트를 **통과하지 못한다**는 전제로 리뷰한다.

## 0. 한 줄 진단

**"잘 만든 학습 분석 플랫폼에 LLM을 덧입힌 문서. 현재 상태로는 회장님이 '우리 팀도 2022년에 비슷한 PoC 했어' 라고 할 가능성 70%."**

문서가 부실해서가 아니다. **개념 수준이 2020~2022년 에듀테크 AI의 평균치에 머물러 있어서**, 2026년에 일본 대학 시장 1티어가 보기엔 **새롭지 않기 때문**이다.

## 1. 시나리오별 AI-native 성적표

판정 기준:
- **AI-native (A)**: 학습 loop이 있고, 개별 맥락에 적응하며, 불확실성을 명시적으로 다룸
- **Mixed (B)**: 일부 AI 요소는 있으나 핵심은 집계·규칙
- **Rule-based (C)**: z-score·카운팅·단순 클러스터링에 LLM 한 줄 얹음

| # | 시나리오 | 판정 | 진단 |
|---|---|---|---|
| S01 | 난관 구간 탐지 | **C+** | 5초 bucket · z-score · HDBSCAN · LLM 요약. **2020년 Panopto·Degreed가 이미 비슷한 PoC 공개함.** 진짜 AI 요소는 "LLM이 3~5건 합성 요약" 한 줄뿐. |
| S02 | 복습 선수 찾기 | **B-** | DKT는 AI 맞는데, D-2에서 **"MVP BKT, 1학기 후 DKT"** 라고 명시함 → 즉 데모 시점엔 1994년 BKT. 망각 곡선도 개인화인지 공통인지 불명. |
| S03 | 질문 쏠림 | **C** | 임베딩 + HDBSCAN + 카운팅. AI 비중 낮음. 정작 "재설명"은 **교수자가 해야 함** — LLM이 해줘야 "AI답다". |
| S04 | 몰아치기 Nudge | **C+** | Risk score + 3단계 nudge. 나쁘진 않은데, **어떤 nudge가 이 학생에게 효과 있었는지 학습하는 loop이 없음** (contextual bandit 부재). |
| S05 | 실시간 강의 | **B** | 히트맵 · 익명 버튼은 UX 훌륭. 하지만 AI 역할은 집계뿐. 교수자를 **실시간으로 돕는 AI** (음성·슬라이드·학생응답 동시 이해)가 없음. |
| S06 | Liner Mode | **C-** | 이건 그냥 PWA UX. AI가 있다면 TTS 정도. **"오늘 통학시간용 팟캐스트"를 학생의 오늘 약점·일정·선호 기반 개인 생성**까지 가야 AI. |
| S07 | Shukatsu Forge | **A-** | **이 문서의 왕관 보석.** 학습 궤적 → 서사 생성 → CLR 검증 링크. 일본 就活 문화·CLR 표준·LLM 생성을 정확히 결합. 청중이 진짜 반응할 지점. |
| S08 | Zemi Hub | **B** | Whisper + 임베딩 + LLM 요약. 평범. **"제미 연구 조교 에이전트"** (상주 + 진척 추적)까지 가야 차별화. |
| S09 | 채점 drift | **B+** | 시간대 drift + kNN 유사 답안 추천. 공정성 angle은 흥미로움. 다만 drift "교정"이 **규칙적 보정**인지 **교수자별 학습**인지 모호. |
| S10 | Early Warning | **C+** | GBT + SHAP. 2018년 모델. **SHAP는 "왜"만 알려주지 "무슨 개입이 효과 있을지"를 모름** — causal forest / uplift 모델이 없으면 처방이 아니라 진단에 그침. |

**요약**: 10개 중 AI-native 1개 (S07), Mixed 3개, Rule-based 6개. **회장님이 보는 체감은 "이건 분석 플랫폼이지 AI 플랫폼이 아니네"**.

## 2. "진짜 AI다" 감을 주려면 — 시나리오별 업그레이드안

### S01 업그레이드 — **"난관 해소 에이전트"**
- 현재: 교수자에게 "이 구간이 난관입니다" 알림 + 개선안 텍스트.
- 업그레이드: 감지 순간 **AI 에이전트가 3가지 대안 5분 미니 강의 스크립트를 즉석 생성** (① 비유 중심 ② 예제 중심 ③ 시각화 중심) → 교수자 선택·편집 → **TTS+슬라이드 자동 합성 영상** 5분 내 배포 → 다음 코호트에 A/B로 효과 자동 측정 → 결과가 해당 강의의 permanent improvement로 편입.
- **데모 impact**: "AI가 문제를 찾는다"에서 "AI가 해결까지 한다"로 넘어감. 회장님이 "오" 하는 순간.

### S02 업그레이드 — **GKT (Graph Knowledge Tracing)**
- 현재: 개념별 BKT (개념 간 독립).
- 업그레이드: L5 개념 그래프를 **GNN + Transformer Knowledge Tracing**으로 학습. 한 개념의 학습이 인접 개념 숙달에 주는 전이 효과를 **학습**. 망각 곡선도 학생별·개념별로 개인화(개인 forgetting rate를 Bayesian inference로 추정).
- **Why now**: 2023~2025 SAKT·AKT·DKVMN 후속 연구들이 이미 구현 가능. "개별 개념별 BKT"는 회장님한테 "그건 구식이야" 맞음.

### S03 업그레이드 — **Generative co-answer**
- 현재: 쏠린 질문 → 교수자가 5분 재설명 해야 함.
- 업그레이드: 질문 쏠림이 감지되면 AI가 **"5분 재설명 스크립트 초안"** + **"즉석 퀴즈 3문항 초안"** + **"학생이 쓴 질문 원문을 교수자 말투로 변환한 FAQ 한 페이지"**를 자동 생성 → 교수자는 검토·수정·게시. 1클릭에 생성→수정→게시.
- **CTA mock**: `[이 질문에 대한 재설명 자료 지금 생성]` (5초 내 초안)

### S04 업그레이드 — **Contextual Bandit Nudge**
- 현재: 3단계 에스컬레이션, 누구에게 뭐가 통하는지 학습 안 함.
- 업그레이드: 각 학생의 과거 nudge 반응을 **contextual bandit (LinUCB / Thompson Sampling)**로 학습. "이 학생은 아침 8시 Soft Pink 톤에만 반응함" 같은 개인화 policy. Omotenashi tone도 자동 선택.

### S05 업그레이드 — **Lecturer Assistant (Ambient AI)**
- 현재: 교수자 대시보드 알림.
- 업그레이드: 교수자가 수업 중 착용하는 **무선 이어피스 또는 스마트워치** + 강의실 대형 화면.
  - **음성 인식**: 교수자 강의 오디오 → 실시간 "어느 개념을 설명 중인지" 추적
  - **슬라이드 인식**: OCR + concept mapping
  - **학생 신호**: 익명 "이해 안됨" + 즉석 퀴즈 + 슬라이드 체류
  - **출력**: 이어피스에 나직한 음성 — *"3명이 카이제곱에서 멈췄습니다. 슬라이드 12로 돌아가는 것을 권장합니다."*
- **이게 "AI-native demo" 의 정수.** 회장님이 영상 보면 그 자리에서 "계약 가능성"이 확 올라감.

### S06 업그레이드 — **Personalized Audio Capsule (Generative)**
- 현재: 할당된 학습 자료의 TTS.
- 업그레이드: **오늘의 팟캐스트를 매일 밤 개인 생성.** 구성:
  - "어제 시청 로그 · 최근 약점 · 오늘 수업 일정 · 다가오는 과제" 조합
  - NotebookLM 스타일 두 명의 호스트가 대화
  - 학생 본인이 직접 쓴 노트·필기도 인용 ("3월 12일 네 노트에 '표준편차 헷갈린다' 라고 적었지?")
- **다른 회사 못 하는 이유**: xAPI Store가 있어야 가능. 즉 **표준 + 생성의 결합이 차별화의 엔진**이라는 메시지로 연결.

### S08 업그레이드 — **Zemi Research Copilot**
- 현재: 미니츠 자동화 + Inheritance Graph.
- 업그레이드: 제미별 상주 **연구 에이전트**.
  - 매 주 학생 연구 진척(커밋·노트·인용 추가)을 감지
  - 인용 그래프 + 과거 제미 산출물 + arXiv·J-STAGE 새 논문 교차 체크
  - "이 방향으로 가면 선배 A가 2023년에 막힌 지점에 도달합니다" 경고
  - **다음 회의 전날 에이전트가 토론 안건 초안을 생성** (미니츠가 아니라 anticipation)

### S10 업그레이드 — **Uplift Modeling for Intervention**
- 현재: GBT 위험 스코어 + SHAP.
- 업그레이드: **Causal Uplift Forest**. "이 학생에게 1:1 면담을 하면 리스크가 얼마나 내려가는가"를 학생별로 예측. **동시에 "Pace Coach 발송" · "무개입" 비교**. SHAP는 "왜 위험한가"만 알려주지만 uplift는 "무슨 개입이 이 학생에게 효과적인가"를 알려준다.

## 3. 빠진 "AI-native" 시나리오 신규 제안 (5개)

### S-new1. Lecture Design Co-Pilot (강의 설계 공동 저자)
- **페인**: 교수자는 강의 준비를 고립해서 한다. 전 학기 코호트 지식 상태를 모르고 시작함.
- **AI**: 강의 준비 모드 진입 시 자동으로:
  - 이번 코호트의 개념 숙달 히트맵 (L3 from previous weeks) 상단 고정
  - 슬라이드 아웃라인 초안 제안 (이전 학기 성공 강의 패턴 학습)
  - 가장 어려운 개념에 대한 **대안 3종** (비유 / 예제 / 시각화) 스크립트 자동 생성
  - 즉석 퀴즈 4문항 초안, 이 코호트에 대한 **예측 정답률** 표기
- **왜 진짜 AI-native**: 이건 분석이 아니라 **공동 창작**. LLM이 메인 롤.
- **CTA**: `[이 강의 초안 생성]` · `[이 개념의 대안 설명 3종 생성]` · `[예상 난관 슬라이드 미리 보기]`

### S-new2. Learning Companion Agent (상주 학습 동반자)
- **페인**: 학생은 AI 챗봇이 아니라 **늘 함께하는 스터디 버디**를 원한다. 현재 시나리오는 모두 "학생이 행동하면 AI가 반응" — reactive.
- **AI**: 학생별 **persistent agent with episodic memory**.
  - 매일 밤 에이전트가 **학생의 그날 학습을 reflect**하고 내일 계획을 준비 (OpenAI Operator / Claude Agent 스타일)
  - 아침 홈 진입 시: *"어제 '분산' 에서 3번 되감으셨어요. 오늘 10분 먼저 복습하고 시작할까요? 제가 3분짜리 요약 준비해뒀어요."* (generative + proactive)
  - 학생이 며칠 잠수하면 오토 메시지: *"괜찮으세요? 무리되시면 오늘 마감을 2일 뒤로 조정해볼 수 있어요. 교수님께는 제가 정중하게 사정 말씀드릴 수도 있어요."*
- **차별 포인트**: 현재 S04의 nudge는 "시스템 규칙". 이건 **에이전트가 학생을 기억하고, 성장을 추적하고, 적극적으로 개입**.

### S-new3. Counterfactual What-If Panel (반사실 시뮬레이션)
- **페인**: 학과장·교수가 교육과정·일정·자료 조정의 효과를 미리 알 수 없음. 의사결정이 사후약방문.
- **AI**: Causal DAG + doubly-robust estimator로 학과 단위 시뮬레이션.
  - *"만약 '확률론' 강의를 주차 3 → 주차 1로 앞당기면?"* → "통계학 II 중간 정답률 +6.2% 예상 (95% CI: 3.1~9.8%)"
  - *"만약 과제 마감을 금요일 23:59 → 월요일 09:00로 바꾸면?"* → "마감 3시간 내 제출 비율 -34%, 평균 품질 +14%, 포럼 질문 +22%"
- **왜 AI-native**: 이건 회장님이 **처음 볼 가능성 높음**. 교육 데이터에 causal inference 제대로 붙인 상용 LMS 드묾.
- **데이터 요구**: 과거 다학기 데이터 + intervention log. 이 시나리오가 존재한다는 것 자체가 **"우리가 데이터를 장기 관점으로 쌓고 있다"**는 증명.

### S-new4. Multimodal Problem-Solving Coach (수식·필기 인식)
- **페인**: 수학·물리·화학·코딩 문제는 **풀이 과정**이 핵심. 현재 LMS는 제출 결과만 본다. 중간에 틀려도 모름.
- **AI**: 태블릿 스타일러스 / 카메라 입력.
  - 수식 필기 → 실시간 MathOCR (LaTeX 변환)
  - 각 단계별 대수 검증 → 첫 오류 발견 즉시 **inline hint** (전체 답이 아니라 그 **한 단계만**)
  - *"3번째 줄에서 ±가 빠진 것 같아요. 근의 공식의 ± 기호를 놓치셨을까요?"*
  - 코딩: 에러 없이 동작해도 **비효율적 패턴** 감지 → "이 부분은 O(n²)이에요. O(n log n)으로 바꿔볼래요?"
- **데모 파워**: 손글씨 쓰는 순간 AI가 실시간 반응 = **AI-native임을 그림으로 증명**.

### S-new5. Institutional Curriculum Optimizer (커리큘럼 자가진화)
- **페인**: 교육과정은 4년에 한 번 위원회에서 고치고, 그 사이 데이터는 사장됨.
- **AI**: 전 학과 다학기 누적 데이터로 causal graph 학습.
  - "Capstone 성공을 가장 잘 예측하는 3학년 과목 Top 5"
  - "신뢰구간 개념이 통계 I·통계 II·계량경제 3과목에서 중복 설명되는데 한 과목만 잘 가르치면 나머지 효과 +7%"
  - "다음 학기 '기계학습 개론' 수강생 예상 분포를 pre-requisite 달성률로 시뮬레이션 → 교재 난이도 조정 제안"
- **AES 맥락**: 학과 / 학교 / 아시아 메타-비교로 자연 확장. OneEdutechKorea 컨소시엄의 **대규모 표준 데이터 자산**이 전제.

### (Bonus) S-new6. Bidirectional Instructor-AI Teaching (교수자가 AI를 가르친다)
- **페인**: AI는 늘 교수자에게 제안한다. 거꾸로 **교수자가 AI를 가르칠** 채널이 없음.
- **AI**: 교수자 프로필에 "교수법 철학 선언":
  - *"나는 '암기보다 유도'를 중시한다. 공식 제시 전에 반드시 2개의 반례를 먼저 보여준다. 어려운 개념일수록 역사적 맥락을 먼저 설명한다."*
  - 이 선언이 시스템 프롬프트로 흡수되어 **이 교수의 모든 학생 측 AI 출력이 이 철학을 반영**.
  - 한 학기 학생 반응으로 선언 자체의 효과도 측정 → 교수에게 피드백.
- **회장님 반응 예상**: "AI가 교수마다 다르게 행동한다는 건 처음 보네." 차별화 강력.

## 5. 데이터·표준 디펜시블리티 강화 (조용상 교수 관점)

### 5-1. xAPI 심화

**지적**: §3-1 공통 헤더에 `xapi_verb` 만 있고, **ADL xAPI Profile 선언이 없다**. 이 상태로는 LRS interoperability 주장이 약함.

**제안**:
- **`edulms.ai-x` xAPI Profile** 을 공식 발행 (JSON-LD). 모든 verb·activity type·extension 정의.
- IMS Global Caliper Analytics와의 **cross-walk 테이블** 제시 (둘 중 하나만 쓰는 게 아니라 양쪽 다 대응한다는 메시지).
- `context.extensions["https://ai-x.jp/xapi/confidence"]` 같은 **AI 관련 custom extension** 정의 — confidence, evidence_count, model_version.

### 5-2. CASE 연동 구체화

**지적**: §3-2에서 L5를 "CASE 표준" 이라고만 씀 (line 671). **CASE Registry와 실제 어떻게 interact하는지 기재 없음.**

**제안**:
- L5 `concept_node` 테이블에 `case_identifier` (GUID) + `case_uri` 필드.
- CASE Registry API (CFItem, CFAssociation) 연동 엔드포인트:
  - `GET /api/v1/case/cfitems/{identifier}`
  - `POST /api/v1/case/import` (외부 registry에서 pull)
- 일본 문부과학성 MEXT 교과 표준과 CASE의 매핑 전략 — 기존 매핑이 없으면 **우리가 최초로 만들겠다** 제안 가능.

### 5-3. CLR 전면 도입

**지적**: §S07에만 CLR 등장. 회장님 관점에서 "왜 CLR가 전체 철학이 아니라 한 시나리오 부속이냐?" 의문.

**제안**:
- **모든** 과제·평가·프로젝트·피어평가·출석·포럼 기여·자격 배지가 **자동으로 CLR entry**로 편입되는 파이프라인을 §3-5 아키텍처에 박음.
- 레코스의 배지 인증 + CLR → **"학습 이력의 국제 이동성"** 을 공동 발표자 레버리지로 강조.
- 학생이 졸업 시 CLR wallet을 **대학·외부 서비스·취업 기업**에 모두 공유 가능.

### 5-4. EDU-API / Ed-Fi / OneRoster 위치 정리

**지적**: §3-5에 LTI 1.3 · xAPI · Webhook만 언급. **OneRoster (학사 연동 표준) · Ed-Fi (데이터 표준 풀스택) · EDU-API (한국 표준)** 가 빠짐.

**제안**:
- 표준 스택을 3 레이어로 구분:
  - **행동 데이터**: xAPI + Caliper (이 문서 중심)
  - **학사 데이터**: OneRoster 1.2 (수강·성적·학기) · Ed-Fi (K-12 호환)
  - **역량·이수 증명**: CLR + Open Badges 3.0 (레코스 역할)
  - **교육과정·역량**: CASE + 교육과정 표준
- 각 layer에 대해 **"어떤 시나리오가 이 표준을 실제로 소비하는지"** 매핑 표 1장 추가.
- 한국 EDU-API (교육부 표준) 대응은 한국 자회사 레코스를 통한 시장 확장 스토리로 연결.

### 5-5. 이벤트 카탈로그 누락 카테고리

- **Agentic actions**: `agent.plan.create` / `agent.plan.execute` / `agent.plan.revise` — 상주 에이전트 동작 로그
- **Generative outputs**: `content.generated.draft` / `content.generated.publish` — AI가 만든 콘텐츠의 라이프사이클
- **Causal interventions**: `experiment.intervention.apply` / `experiment.outcome.measure`
- **Multimodal**: `input.handwriting.stroke` / `input.voice.utterance` / `input.camera.capture`
- **Cross-system**: `interop.clr.emit` / `interop.case.map` / `interop.lti.launch`

## 6. 재작성이 필요한 "뜬소리" 문장

### 6-1. "Evidence + Reasoning + Action 3종 강제" (line 30, 785)
- **문제**: "강제"의 근거는 §3-3 CI 체크 3줄뿐 (line 707-710). 강제라는 말을 쓰려면 governance 위원회·외부 감사 loop까지 있어야 "강제" 느낌.
- **대안**: **"AI Governance Pipeline: 4-Gate"**
  > 모든 AI 출력은 ①스키마 CI ②evidence·confidence 자동 검증 ③월간 Bias·drift 리뷰 위원회 ④Mei-waku 피드백 재학습 — 4게이트를 통과해야 배포. 외부 감사 리포트 연간 공개.

### 6-2. "BKT/DKT 기반 개념별 숙달 확률" (line 33)
- **문제**: 테이블은 마치 DKT가 운영 중인 것처럼 읽히는데, D-2에서 "MVP는 BKT, 이후 DKT"라 함 (line 794). 청중이 디테일 물으면 꼬임.
- **대안**: **"Bayesian Knowledge Tracing (MVP) → Graph Knowledge Tracing (P2) — 코호트 데이터가 쌓이는 즉시 전환, 현재 운영 중인 모델을 학생·교수에게 투명하게 표시"**. 로드맵까지 투명한 게 신뢰도 높음.

### 6-3. "일본 문화 특화" (line 47)
- **문제**: Omotenashi 단어만 있고 근거 부재. 회장님: *"日本文化に特化って、Omotenashiって言葉以外に何があるの?"*
- **대안**: 구체적으로:
  - **Meiwaku Awareness**: 경고 단어 금지 + 낙인 방지 UX 3중 강제
  - **Honne/Tatemae 인식 텍스트 생성**: 일본어 피드백의 직설 수위를 자동 조정
  - **학사 특성 대응**: 前期/後期 · 履修登録 · 通年科目 · 集中講義 · 卒論ゼミ 구조를 스키마 레벨에서 1급 시민으로 지원
  - **敬語 수준 자동 제어**: 학생→교수 메시지, 교수→학생 피드백, Shukatsu 문서 모두 경어 레벨 3종(です・ます/である/敬語) 선택·자동
  - **설문·실증**: 일본 대학 3곳 UX 설문 n≥300 으로 Omotenashi principle을 정량 검증

### 6-4. "GNN 추천 → 교수 승인" (D-1, line 794)
- **문제**: GNN이 §3-5 모델 스택에 없음. 뜬구름.
- **대안**: 삭제하거나, §3-5에 GNN (예: L5 개념 그래프 위에서 학습) 정식 편입. 이 경우 S02 GKT와 공유 인프라로 설명.

### 6-5. "낙인 방지 UX가 UX 레벨에서 강제" (line 45)
- **문제**: "강제"의 기술적 구현 미기재. 브라우저 확장으로 무력화 가능한 수준이면 "강제"가 아님.
- **대안**:
  - 컴포넌트 레벨: 학생 명단 view는 `<RiskIndividualView purpose required />` 컴포넌트로만 접근 가능, purpose 입력 시 서버측 audit log 기록 후 token 발급.
  - 서버 레벨: 개별 학생 위험 스코어 API는 purpose token 없이 401.
  - 브라우저 bypass 방지: 클라이언트는 집계만 받고, 개인 데이터는 purpose token이 있어야만 페이로드 반환.

### 6-6. "망각 곡선(concept_forgetting.predicted_retention_today) < 40%" (line 348)
- **문제**: 개인화인지 공통인지 불명. 공통이면 Ebbinghaus 1885. 회장님: *"それ、140年前の論文ですよね?"*
- **대안**: **"개인별·개념별 forgetting rate Bayesian posterior. 사용자 응답 이벤트 10건 이상에서 수렴. 콜드 스타트는 개념 그래프 neighbor의 pooled prior로 시작."**

### 6-7. "적용 세그먼트 재시청률 -30%" 등 (line 76, 294)
- **문제**: A/B 방법론 미기재. "누구 vs 누구" (같은 학생 전후? 같은 영상 다른 학생? 다른 학기 코호트?) 모호.
- **대안**: **"cluster-randomized trial at section × content level, 2주 관측, pre-registered outcome, two-sided 95% CI"**. 표준 대응 코드와 함께.

1. **BKT vs 즉시 GKT**: 나는 "MVP부터 GKT 선로드맵 공개"가 청중 기대 수준에 맞다고 봄. Codex는 "BKT 유지, 대신 다른 차별화"를 제안할 수도 있음.
2. **Ambient AI 이어피스**: 데모 파워는 강하지만 구현 난이도·개인정보 이슈 큼. 기술적으로 권장 가능한지 교차 검증 필요.
3. **S10의 Uplift vs Policy learning**: Uplift Forest가 맞는지, RL policy가 맞는지 정책 레벨에서 더 나은 선택이 있는지.
4. **"Ambient AI 영상"을 데모 1순위로 할지, Shukatsu Forge를 1순위로 할지**: 일본 청중 효용 vs 글로벌 임팩트 사이 판단.
5. **causal inference 시나리오 (S-new3)**: 데이터가 없는 MVP 단계에서 회장님께 약속해도 될지, 아니면 P3~P4 로 미뤄야 할지.
6. **"일본 문화 특화" 정의**: Omotenashi를 넘어선 구체 목록에서 무엇을 빼고 무엇을 더해야 하는지.
7. **레코스·위키드스톰·조용상 교수·NetLearning 의 4자 role split**: 이 문서에 이 정치적 구조가 반영돼야 하는가, 별도 문서가 맞는가.
