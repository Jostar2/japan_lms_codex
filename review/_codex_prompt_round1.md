## 리뷰 우선순위 (이 순서대로 예리하게)

### 1순위 — "이건 진짜 AI다" vs "LLM 뿌린 대시보드다" 판별
각 시나리오(S01~S10)를 다음 기준으로 분류:
- **AI-native (진짜)**: 학습 loop이 있고, 일반화하며, 불확실성을 다루고, 개별 맥락에 적응함
- **Rule-based (대시보드)**: 임계값·z-score·카운팅·단순 클러스터링에 LLM 출력을 얹은 수준
- 각 분류에 대해 **왜 그렇게 판단했는지 + 어떻게 진짜 AI-native로 업그레이드할지 구체안**
- 특히 S01(난관 구간), S02(복습), S04(몰아치기), S10(Early Warning)은 고전적 규칙 기반 냄새가 강함 → 무엇을 어떻게 바꿔야 "와, 이 팀 진짜 AI 하네" 소리가 나올지

### 2순위 — "이게 빠졌네" AI-native 시나리오 신규 제안
회장님이 "오, 이건 처음 보네" 할 만한 걸 3~5개. 각각:
- 한 줄 정의
- 페인포인트 (actor별)
- 감지 → 추론 → 출력 파이프라인 (구체적 모델·신호·임계값까지)
- 교수자/학생 화면·CTA mock
- 기존 시나리오와 공유하는 데이터 자산

**탐색 방향 힌트** (얽매이지 말고 더 좋은 게 있으면 제시):
- **Agentic**: 학생 대신 학습 계획을 세우고 · 실행하고 · 회수하는 상주 에이전트 (단발 추천이 아니라 "진행형")
- **Generative co-creation**: LLM이 교수자/학생과 함께 콘텐츠를 **공동 생성** (기존 텍스트를 보여주는 게 아님)
- **Multimodal**: 필기·음성·표정·코드·수식 동시 이해
- **Causal/Counterfactual**: "만약 이 개입을 했다면 성적이 어떻게 변했을까" 반사실 시뮬레이션
- **Real-time adaptation**: 학생 응답 1건마다 난이도/경로가 즉시 재계산되는 IRT+RL 혼합

### 3순위 — 화면 · CTA · 인터랙션 구체화
"이 순간이 AI답다"고 느껴질 구체적 UI 모먼트를 **mock UI 박스**로 그려서 제안:
- 교수자/학생이 **무심코 하는 행동이 AI 학습으로 변환되는 순간**
- CTA 문구는 실제 일본어 / 한국어 둘 다 (청중이 일본인이므로 일본어 문구가 있으면 +α)
- 현재 §01-H의 xAI 카드를 어떻게 더 차별화할지

### 4순위 — 데이터·표준 디펜시블리티 강화
조용상 교수님이 옆에서 듣고 계심. 표준 관점에서 약점:
- **xAPI Statement** 설계가 실제로 의미 있게 쓰이는가, 아니면 로그 저장소 장식용인가
- **CASE** (1EdTech Competencies and Academic Standards Exchange) 연동 논리가 모호함 — L5 개념 그래프와 CASE를 어떻게 매핑할지
- **EDU-API / Ed-Fi** 와의 관계는? (§3-1에 없음)
- **CLR (Comprehensive Learner Record)** 이 S07에만 등장하는데, 전체 시나리오에 어떻게 관통시킬지
- **OneRoster · LTI Advantage · Caliper Analytics** 와 xAPI의 역할 분담을 회장님 앞에서 설명할 수 있는 수준으로 써야 함
- 이벤트 카탈로그(§3-1)에 **빠진 카테고리** 있으면 지적 + 추가 제안

### 5순위 — "뜬소리 문장" 재작성
원문에서 세미-버즈워드/모호한 문장 최소 5개 인용 → 왜 문제인지 → 구체 대안 문장

### 6순위 — 전체 총평 (회장님 시뮬레이션)
- 지금 이 문서를 20분 프레젠으로 들으면 회장님이 **어디서 "그건 이미 다 해봤어"** 라고 할지 3지점
- 회장님이 **"오, 이건 다르네"** 라고 할 지점 3개 (원문에 이미 있으면 유지)
- 프레젠 구조(도입→증명→파일럿 제안) 관점에서 지금 문서의 치명적 약점
